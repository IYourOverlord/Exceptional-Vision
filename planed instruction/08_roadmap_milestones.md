# Roadmap и критерии готовности

## Этап 0 — Юридическая и организационная база
- [ ] Выбрана лицензия проекта (MIT/LGPL-3.0), файл `LICENSE` добавлен.
- [ ] README содержит формулировку независимости от Voxy/DH (см.
      `01_legal_constraints.md`).
- [ ] Настроен Gradle-проект Forge 1.21.1 MDK, мод грузится пустым.

**Критерий готовности:** мод устанавливается, ничего не делает, лог
чистый, лицензия и README на месте.

## Этап 1 — Независимое чтение мира
- [ ] `RegionFileReader` читает `.mca`, распаковывает NBT секции.
- [ ] `WorldDirectoryWatcher` реагирует на новые/изменённые регионы.
- [ ] Команда полного импорта существующего мира (аналог очереди по всем
      регионам) — см. `05_world_data_ingestion.md`.
- [ ] Юнит-тест: подать заранее сгенерированный тестовый мир (в т.ч.
      регион, сгенерированный Chunky) и убедиться, что все чанки читаются
      без загрузки в игровой `ServerLevel`.

**Критерий готовности:** консольная утилита/тест печатает высоты и
материалы верхних блоков для произвольного региона без запуска
Minecraft-сессии.

## Этап 2 — LOD-структура и даунсемплинг
- [ ] Реализован quadtree (`LodNode`) с уровнями детализации.
- [ ] Даунсемплинг колонки блоков в высоту + материал + средний цвет.
- [ ] Greedy meshing, генерирующий `PackedQuad` (см. `03_data_formats.md`).
- [ ] Даунсемплинг выполняется в пуле потоков, не блокирует I/O и рендер.

**Критерий готовности:** для тестового региона строится корректное
дерево узлов с ожидаемым количеством квадов на каждом уровне.

## Этап 3 — Дисковый кэш
- [ ] Формат `nodes.bin`/`quads.bin`/`index.json` реализован (см.
      `06_disk_cache_format.md`).
- [ ] При повторном запуске мод не пересчитывает уже обработанные
      регионы (проверка по `mtimeMs`/hash).
- [ ] Инкрементальное обновление при изменении отдельного региона.

**Критерий готовности:** повторный запуск на том же мире occurs
значительно быстрее первого (LOD загружается из кэша, а не считается
заново) — замерить и зафиксировать разницу во времени.

## Этап 4 — GPU-пайплайн (базовый, без indirect count)
- [ ] SSBO-буферы созданы, заполняются из кэша при старте.
- [ ] Compute shader culling реализован, читает `drawCount` обратно на
      CPU для явного вызова `glMultiDrawArraysIndirect` (fallback-путь
      для GPU без OpenGL 4.6, см. `04_gpu_pipeline.md`).
- [ ] Vertex/fragment shader генерируют геометрию квадов из SSBO.

**Критерий готовности:** LOD-геометрия видна в игре на дистанции, где
ванильный рендер уже не отображает чанки; FPS не проседает
катастрофически при дистанции ~1000-2000 блоков.

## Этап 5 — Полный GPU-driven путь
- [ ] `glMultiDrawArraysIndirectCount` задействован на GPU с поддержкой
      OpenGL 4.6, без readback счётчика на CPU.
- [ ] Persistent mapped buffers для стриминга новых данных без стоплов.
- [ ] `GpuCapabilities.detect()` корректно переключает между полным и
      fallback-путём в зависимости от возможностей драйвера.

**Критерий готовности:** профилирование (RenderDoc/Nsight) показывает
GPU-bound профиль вместо CPU-bound, время CPU на кадр в LOD-рендере
стабильно и не растёт с увеличением render distance.

## Этап 6 — Стыковка с ближним рендером
- [ ] Mixin-интеграция в `LevelRenderer` (см. `07_forge_integration.md`).
- [ ] Отсутствие видимого шва/z-fighting на границе ближний/LOD рендер.
- [ ] Fade/dithering на границе для визуального сглаживания.

**Критерий готовности:** визуальное сравнение скриншотов границы до/
после fade показывает отсутствие резкого разрыва геометрии.

## Этап 7 — Конфигурация и UX
- [ ] Конфиг: дистанция LOD, включение/выключение GPU-driven culling,
      качество LOD-палитры.
- [ ] Сообщение игроку при неподдерживаемом GPU вместо тихого краша.
- [ ] Команда ручного импорта существующего мира из GUI/чата.

**Критерий готовности:** мод пригоден для передачи внешним тестерам без
необходимости объяснять консольные команды.

## Этап 8 — Полировка и релиз
- [ ] Профилирование памяти VRAM на больших мирах (десятки тысяч
      регионов).
- [ ] Тестирование на нескольких видеокартах/драйверах (NVIDIA, AMD,
      Intel — совместимость с OpenGL 4.3/4.6 путями).
- [ ] Документация для пользователей, лицензия, changelog.

**Критерий готовности:** стабильная бета-версия, готовая к публикации
на Modrinth/CurseForge под собственным именем и лицензией.
