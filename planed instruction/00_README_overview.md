# LOD Distant Rendering Mod — техническая спецификация проекта

Этот набор файлов описывает архитектуру мода для Minecraft Forge 1.21.1,
реализующего GPU-driven рендеринг дальних дистанций (Level of Detail),
аналогичного по функциональности Distant Horizons и Voxy, но являющегося
**независимой реализацией с нуля**.

Используй эти файлы как техническое задание: каждый файл — самостоятельный
модуль спецификации, который можно реализовывать по отдельности.

## Структура файлов

- `01_legal_constraints.md` — юридические рамки проекта. Прочитать первым.
- `02_architecture_overview.md` — общая архитектура пайплайна, потоки данных.
- `03_data_formats.md` — бинарные форматы: упаковка квадов, структура узлов
  quadtree, layout SSBO-буферов.
- `04_gpu_pipeline.md` — GPU-driven рендеринг: compute culling, indirect
  multidraw, генерация геометрии в шейдере.
- `05_world_data_ingestion.md` — независимое от игрового мира чтение
  region-файлов (.mca), включая чанки, сгенерированные сторонними
  инструментами (например Chunky).
- `06_disk_cache_format.md` — формат кэша LOD-данных на диске.
- `07_forge_integration.md` — интеграция с рендер-пайплайном Forge 1.21.1
  через Mixin, без конфликта с ванильным рендером.
- `08_roadmap_milestones.md` — пошаговый план разработки с критериями
  готовности каждого этапа.

## Ключевые технические принципы проекта

1. **GPU-driven, не CPU-driven.** CPU не строит геометрию и не делает
   frustum culling каждый кадр — эти задачи выполняет GPU (compute shader +
   indirect draw). CPU только готовит компактные "сырые" данные.
2. **Независимость от игрового мира.** LOD-данные строятся из чтения
   region-файлов на диске напрямую, а не из чанков, загруженных в текущую
   игровую сессию. Это позволяет отрисовывать территорию, сгенерированную
   заранее (например модом Chunky), даже если игрок там никогда не был.
3. **Минимальное требование:** OpenGL 4.3 (compute shaders) для базовой
   версии, OpenGL 4.6 (`ARB_indirect_parameters`) для полной GPU-driven
   индиректной отрисовки без чтения счётчика на CPU.
4. **Полностью самостоятельная кодовая база.** Ни строки кода не заимствовано
   из стороннего мода с ограничительной лицензией — см. `01_legal_constraints.md`.

## Как использовать эти файлы с ИИ-ассистентом разработки

Рекомендуемый порядок промптов для кодового ассистента (например Claude Code):

1. Передать `01_legal_constraints.md` и `02_architecture_overview.md` —
   попросить сгенерировать структуру Gradle-проекта Forge 1.21.1 MDK.
2. Передать `05_world_data_ingestion.md` — реализовать чтение .mca независимо
   от `ServerLevel`.
3. Передать `03_data_formats.md` и `06_disk_cache_format.md` — реализовать
   даунсемплинг в LOD и сериализацию кэша.
4. Передать `04_gpu_pipeline.md` — реализовать GLSL-шейдеры и Java-обвязку
   через LWJGL.
5. Передать `07_forge_integration.md` — встроить рендер в игровой цикл.
6. Использовать `08_roadmap_milestones.md` как чеклист прогресса.
