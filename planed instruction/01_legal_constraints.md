# Юридические рамки проекта

## Контекст

Проект вдохновлён функциональностью двух существующих модов:

- **Voxy** (MCRcortex) — лицензия "All Rights Reserved", явно указано
  "Do not redistribute". Код закрыт для форков, портов и заимствования.
- **Distant Horizons** (jeseibel) — лицензия **LGPL-3.0-only**, полностью
  открытый исходный код.

## Обязательные правила для разработки и для ИИ-ассистента

1. **Не читать, не копировать, не декомпилировать исходный код Voxy.**
   Никакие фрагменты кода, структуры классов, имена внутренних API или
   комментарии из Voxy не должны попадать в этот проект ни в каком виде.
   Разрешено использовать только публично известные *концепции*
   (LOD-рендеринг, quadtree, greedy meshing, GPU-driven culling) — это
   общеотраслевые техники, не защищённые авторским правом сами по себе.

2. **Distant Horizons (LGPL-3.0) можно изучать как референс.** Если в
   какой-то момент разработки будет использован конкретный фрагмент кода
   или модуль из DH — он должен:
   - сохранять оригинальный copyright и текст лицензии LGPL-3.0;
   - быть выделен в отдельный модуль/файл с явной пометкой источника;
   - не смешиваться "тихо" с остальным кодом без разделения границ модуля.
   Если это создаёт неудобства (LGPL требует, чтобы заимствованная часть
   оставалась LGPL или динамически линковалась) — проще писать с нуля.

3. **Формат region-файлов (.mca) не является объектом авторского права
   в смысле кода.** Это опубликованная спецификация формата данных
   (документация доступна в открытых источниках, например Minecraft Wiki).
   Собственный парсер этого формата, написанный с нуля, полностью легален.

4. **Название и брендинг.** Не использовать имена "Voxy", "Distant
   Horizons" в названии мода или как способ намекнуть на официальную
   связь с ними (это может нарушать нормы добросовестной конкуренции/
   trademark, даже если код независим).

5. **Лицензия проекта.** Рекомендуется MIT или LGPL-3.0 для собственного
   кода — это упрощает получение контрибьюторов (в отличие от ARR-модели,
   на неудобство которой прямо жалуются пользователи в issues Voxy).

6. **Формулировка в README:** "Independent implementation of GPU-driven
   LOD terrain rendering, inspired by the concept popularized by mods such
   as Distant Horizons and Voxy. No code shared with either project."

## Что можно свободно использовать без каких-либо ограничений

- Общеизвестные алгоритмы: greedy meshing, quadtree/octree LOD, frustum
  culling, GPU-driven indirect rendering, compute shader culling —
  это публично задокументированные техники (GPU Gems, доклады GDC,
  спецификации Khronos OpenGL/Vulkan).
- Официальная документация Forge/Minecraft Mixin API.
- Спецификация формата Anvil/.mca (публичная документация).
